## File Descriptions

### Root Files
- **README.md**: Main project documentation with overview and setup instructions
- **requirements.txt**: Python package dependencies (numpy, matplotlib, pandas, etc.)
- **.gitignore**: Files and directories to ignore in version control

### report/
LaTeX files for the final academic paper:
- **main.tex**: Main LaTeX document with ACM template
- **bibliography.bib**: Bibliography entries for citations
- **figures/**: Directory for graphs and diagrams (auto-generated)

### src/
Python source code for implementations:

#### src/greedy/
- **algorithm.py**: Greedy algorithm implementation
  - `GreedyAlgorithm` class with `solve()` method
  - Test data generation
  - Runtime measurement functions
- **experiment.py**: Experimental validation script
  - Runs experiments on various input sizes
  - Generates runtime graphs
  - Saves results to CSV

#### src/divide_conquer/
- **algorithm.py**: Divide-and-conquer implementation
  - `DivideConquerAlgorithm` class with `_divide_conquer()` method
  - Test data generation
  - Runtime measurement and operation counting
- **experiment.py**: Experimental validation script
  - Runs experiments on various input sizes
  - Generates multiple analysis graphs
  - Tracks operation counts

#### src/utils/
- **helpers.py**: Common utility functions
  - Timer decorator
  - Algorithm comparison functions

### experiments/
Generated experimental data:
- **data/**: Optional directory for test datasets
- **results/**: CSV files with experimental results

### docs/
Project documentation:
- **llm_usage_log.md**: **REQUIRED** - Document all LLM interactions
- **problem_ideas.md**: Suggestions for problems to solve
- **checklist.md**: Project progress checklist
- **quickstart.md**: Quick start guide for getting started

## Usage Flow

1. **Setup**: Install dependencies from `requirements.txt`
2. **Choose**: Select problems using `docs/problem_ideas.md`
3. **Implement**: Edit `src/greedy/algorithm.py` and `src/divide_conquer/algorithm.py`
4. **Test**: Run individual algorithm files to test
5. **Experiment**: Run `experiment.py` files to generate data and graphs
6. **Document**: Update `docs/llm_usage_log.md` for all LLM usage
7. **Write**: Complete `report/main.tex` with your findings
8. **Submit**: Generate PDF and submit with code

## Key Features

- **Modular Structure**: Separate implementations for each algorithm type
- **Automated Experiments**: Scripts to generate runtime graphs automatically
- **Professional Report**: ACM-formatted LaTeX template
- **Documentation**: Comprehensive guides and checklists
